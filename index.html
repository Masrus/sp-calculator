Как работает процесс оценки

Оценка идёт по пятиэтапному процессу. После Research становится понятно — это задача или эпик.

1. Research — до 2 SP

Уточнение требований, построение сценария, декомпозиция.

↓ После Research: задача или эпик?

Если эпик — дробишь на подзадачи. Каждая подзадача проходит этапы 3-5 отдельно.

2. Draft — ~6 SP

Черновой макет по всему эпику или задаче, проверка гипотез.

3. Main Flow — 2-16 SP

Финальный макет основного сценария. Для эпика — отдельная задача на каждую часть.

4. Подготовка к разработке — ~4 SP

Corner cases, спецификация, состояния интерфейса.

5. Design Review — 1 SP

Проверка соответствия реализации макетам.

Вкладка Быстрая оценка — для нетипичных задач и фиксов, которые не вписываются в процесс.

Best practices

Калибровка под недельный спринт

При capacity 16 SP на человека, недельный спринт вмещает 32 SP на команду из двух дизайнеров. 16 SP = 5 рабочих дней. Любая задача > 8 SP съедает больше половины недели — подумайте о дроблении.

Используйте эталонную задачу

Выберите задачу, которую вся команда понимает как «среднюю» (например, на 3 SP ≈ 2 дня работы), и оценивайте всё остальное относительно неё.

Включайте время на коммуникацию

В SP должна входить не только «отрисовка», но и время на обсуждение с менеджерами и передачу макетов в разработку.

Не переводите SP в часы напрямую

1 SP ≈ 4-5 часов работы с учётом всего (митинги, правки, коммуникация). Основная цель SP — предсказать объем, который команда может «переварить» за спринт (Velocity).

Дробите задачи на 8+ SP

Если задача оценивается в 8 SP — она съедает всю неделю одного дизайнера. Задачи в 13+ SP вообще не влезают в недельный спринт. Дробите обязательно.

Примеры оценки

Правка иконок → 2 SP (~полдня)

Только Draft (1-2 SP). Research не нужен — задача понятна сразу.

Форма регистрации → 12 SP (~3 дня)

Research (2) + Draft (6) + Подготовка (4) = 12 SP.

Корзина с состояниями → 19 SP (~вся неделя+)

Research (2) + Draft (6) + Main Flow (6) + Подготовка (4) + Review (1) = 19 SP → стоит дробить.

Редизайн раздела → эпик, 3+ недели

Research (2) + Draft (6) = 8 SP — это первая задача.
Затем 3-4 подзадачи: Main Flow часть N (10-16) + Подготовка (4) + Review (1) каждая.

Как дробить эпики

Дробить Main Flow удобно по одному из принципов:

По функционалу

Happy path → Edge cases → Пустые состояния → Ошибки

По экранам

Главный экран → Детальная страница → Форма создания

По платформам

Desktop → Mobile → Tablet
